\leftskip=0pt
\section*{Creating a Feature Branch}
\leftskip=1.5em

\begin{lstlisting}
git checkout a3
git pull origin a3
git checkout -b a3-pN    # replace N with your part number
\end{lstlisting}

Branch names use hyphens (\texttt{a3-pN}), not slashes (\texttt{a3/pN}), because git cannot create \texttt{a3/p3} when \texttt{a3} already exists as a branch.

Work inside your \texttt{pN/} directory. Merge conflicts should be rare --- if they occur, they will be in \texttt{a3/docs/} (shared files).

\leftskip=0pt
\section*{Merging All Parts}
\leftskip=1.5em

The person assembling the final submission should run:

\begin{lstlisting}
git checkout a3
git merge a3-p2 --no-ff -m "Merge P2: architecture changes"
git merge a3-p3 --no-ff -m "Merge P3: context window extension"
git merge a3-p4 --no-ff -m "Merge P4: final nanochat training"
\end{lstlisting}

\texttt{--no-ff} preserves branch history for the reviewer.

\subsection*{Verify}
\leftskip=3em

\begin{lstlisting}
git log --oneline --graph a3
ls a3/    # should show docs/ shared/ p2/ p3/ p4/
\end{lstlisting}

\leftskip=0pt
\section*{Submitting to Course Repo}
\leftskip=1.5em

Ensure \texttt{a3.pdf} is on the \texttt{a3} branch, then:

\begin{lstlisting}
gh pr create --base main --head a3 \
  --title "A3: Nanochat Pre-training"
\end{lstlisting}
